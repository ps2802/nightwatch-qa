# Reports

- `/reports/raw` – Source artifacts (screenshots, HAR, traces). Structure: `reports/raw/<client>/<YYYY-MM-DD>/<artifact>`.
- `/reports/samples` – Sanitized examples (mdBook source + exported PDF) for sales.
- `/reports/templates` – To be added; mdBook theme + Pandoc configs.

## Report outline (default)
1. Executive summary
2. Scorecard (friction scores, severity)
3. Persona journey maps (desktop + mobile)
4. UX critiques (Outline)
5. Copy critiques (Echo)
6. Conversion plays (Funnel)
7. Technical notes (Spin)
8. Quick wins vs roadmap
9. Appendix: evidence links + GitHub export

Everything here should be clonable + reviewable with standard Markdown tooling.
